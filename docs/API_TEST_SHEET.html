1. 로그인 화면 (Login.tsx)
1-1 OTP 설정 조회 POST /api/common/getCommonCodes POST 어댑터 /api/common/getCommonCodes CODE_GROUP: "MOOT001" △ OTP_ENABLED=true일 때만 호출. ref_code=서버IP, ref_code2=DEV/LIVE, name=SharedSecret
1-2 OTP 단독 인증 (선택) POST /api/auth/otp-verify EC2 내부 → RADIUS UDP 1812 USR_ID, OTP_CODE(6자리) △ OTP만 별도 검증 시 사용. RADIUS AccessRequest → AccessAccept/Reject
1-3 통합 로그인 (ID+PW+OTP) — /api/auth/login-with-otp 내부 흐름
Step 1. 로그인 시작 기록 POST /api/auth/login-with-otp → POST 어댑터 /api/system/pm/pmobileLoginApi_1 P_LOGIN_TRX_ID, P_USER_ID, P_NW_TYPE, P_CLIENT_IP, P_USER_AGENT, P_SERVER=EC2_MOBILE, P_API_TYPE=LOGIN △ TRX_ID 형식: userId_yyyyMMddHHmmss
Step 2. CONA ID/PW 인증 → POST 어댑터 /api/login USR_ID, PASSWORD, LOGIN_VIEW=MOBILE, DRM_VERSION=1000, DISCONN_YN △ JSESSIONID 저장. 30s 타임아웃.
Step 3. 로그인 결과 기록 → POST 어댑터 /api/system/pm/pmobileLoginApi_2 P_LOGIN_TRX_ID, P_API_TYPE=LOGIN, P_RESULT_CD=SUCCESS/FAIL, P_RESULT_MSG △ 실패 시 Step 5(API_3)로 직행 → FAIL 반환
Step 4. OTP RADIUS 인증 EC2 → RADIUS UDP:1812 (OTP서버)
→ 어댑터 /api/system/pm/pmobileLoginApi_1 (API_TYPE=OTP)
→ 어댑터 /api/system/pm/pmobileLoginApi_2 (API_TYPE=OTP) 		RADIUS: User-Name, User-Password (MD5 XOR)
OTP서버 IP는 MOOT001에서 동적 로드		 △ OTP_CODE 있을때만. 6자리 검증. 캐시 없음(매 로그인마다 DB조회)
Step 5. 최종 결과 기록 → POST 어댑터 /api/system/pm/pmobileLoginApi_3 P_LOGIN_TRX_ID, P_FINAL_RESULT_CD=SUCCESS/FAIL/OTP_FAIL, P_FINAL_RESULT_MSG △ 성공 시 loginResult + trxId 반환
1-4 레거시 직접 로그인 (OTP 미사용) POST /api/login POST 어댑터 /api/login USR_ID, PASSWORD, LOGIN_VIEW=MOBILE, DRM_VERSION=1000, DISCONN_YN △ OTP_ENABLED=false 시 사용. pmobileLoginApi 감사로그 없음.
1-5 지점 목록 조회 POST /api/customer/equipment/getAuthSoList POST 어댑터 → CONA bean USR_ID △ 로그인 성공 후 AUTH_SO_List 획득
1-6 공통코드 조회 (OTP 설정) POST /api/common/getCommonCodes POST 어댑터 /api/common/getCommonCodes CODE_GROUP: "MOOT001" △ Login.tsx에서 OTP on/off 판단. LIVE/DEV 환경 구분.
2. Activity Log (TSYMO_APP_ACTIVITY_LOG)
2-1 활동 로그 저장 POST /api/system/pm/insertActivityLog POST 어댑터 /api/system/pm/insertActivityLog LOG_TYPE, USR_ID, USR_NM, SO_ID, CRR_ID, ACCESS_TYPE, FROM_VIEW, TO_VIEW, MENU_NM, ORDER_ID, NW_TYPE, LOGIN_TRX_ID, CRT_DTTM △ 10초 배치 or 50건 or 페이지 이탈 시 flush. sendBeacon 우선.
LOG_TYPE: LOGIN 로그인 성공 시 → TO_VIEW=LoginPage Login.tsx logLogin() △ App.tsx handleLogin()에서 호출
LOG_TYPE: LOGOUT 로그아웃 시 → FROM_VIEW=현재경로 App.tsx handleLogout() △ 세션타임아웃(60분) 포함
LOG_TYPE: MENU_CLICK 화면 전환 시 → FROM_VIEW, TO_VIEW, MENU_NM 자동매핑 navigateToView, navigateBack, navigateToMenu, popstate △ 작업완료 콜백, 뒤로가기 버튼 포함
LOG_TYPE: WORK_COMPLETE 작업 완료 시 → TO_VIEW=WorkComplete, ORDER_ID useCompleteWork.ts △ React Query onSuccess에서 호출
3. Debug Log (TSYMO_APP_DEBUG_LOG)
3-1 디버그 로그 저장 POST /api/system/pm/insertDebugLog POST 어댑터 /api/system/pm/insertDebugLog LOG_LEVEL, USR_ID, SO_ID, API_PATH, API_METHOD, API_STATUS, API_DURATION, ERROR_MSG, REQ_BODY, RES_BODY, STACK_TRACE, PAGE_VIEW, NW_TYPE, LOGIN_TRX_ID △ 동일 배치 방식 (10초/50건)
API 에러 (logApiError) apiService.ts 인터셉터 — HTTP 4xx/5xx 자동 캡처 LOG_LEVEL=ERROR, API_PATH, API_STATUS, API_DURATION △ timeout(408), network error(0) 포함
느린 API (logSlowApi) apiService.ts 인터셉터 — 응답시간 초과 시 자동 LOG_LEVEL=WARN, API_DURATION(ms) △ 임계값 초과 시
전역 에러 (GlobalError) window.addEventListener('error') → logRuntimeError LOG_LEVEL=ERROR, ERROR_MSG, STACK_TRACE △ App.tsx 전역 핸들러
미처리 Promise (UnhandledRejection) window.addEventListener('unhandledrejection') → logRuntimeError LOG_LEVEL=ERROR, ERROR_MSG, STACK_TRACE △ App.tsx 전역 핸들러
React 에러 (ErrorBoundary) ErrorBoundary componentDidCatch → logRuntimeError LOG_LEVEL=ERROR, 컴포넌트 스택 △ ErrorBoundary.tsx
장비 프리로드 성공 WorkProcessFlow.tsx — 장비 데이터 조회 완료 LOG_LEVEL=INFO, WRK_ID, WRK_CD, 장비수량(contract/tech/customer/removed) △ 작업 진행 화면 진입 시
장비 프리로드 실패 WorkProcessFlow.tsx — 장비 데이터 조회 오류 LOG_LEVEL=ERROR, WRK_ID, WRK_CD, 에러메시지, STACK_TRACE △ catch 블록
작업완료 확인 팝업 WorkCompleteSummary.tsx — 작업완료 최종 확인 데이터 LOG_LEVEL=INFO, 작업유형, 고객관계, 망구분, 설치유형, 장비목록(설치/철거), 메모 △ 관리자가 main 머지 시 추가 (ac0ad0bb)
작업완료 실패 useCompleteWork.ts onError — 완료 API 실패 LOG_LEVEL=ERROR, WRK_ID, 에러메시지, STACK_TRACE △ React Query onError

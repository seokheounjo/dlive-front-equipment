# 📱 Mobile CONA Frontend

딜라이브 현장 기사용 모바일 웹 애플리케이션 - 실제 딜라이브 JSON API 연동

## 🎯 프로젝트 개요

기존 MiPlatform 기반 Mobile CONA를 React/TypeScript 기반 모던 웹 애플리케이션으로 전환한 프로젝트입니다.  
실제 딜라이브 운영 서버와 JSON API로 연동하여 더미 데이터가 아닌 **실제 DB 데이터**를 사용합니다.

## ✅ 현재 구현 완료된 기능

### **1. 사용자 인증**
- ✅ **실제 로그인**: 딜라이브 계정으로 로그인 (`/api/login`)
- ✅ **세션 관리**: 쿠키 기반 세션 유지
- ✅ **오류 처리**: 로그인 실패 시 구체적 메시지 표시

### **2. 작업 관리**
- ✅ **작업조회**: 실제 DB에서 작업지시 목록 조회 (`/api/work/directions`)
- ✅ **작업취소**: 실제 작업취소 API 연동 (`/api/work/cancel`)
- ✅ **날짜 필터**: 한 달 단위 기본 조회, 사용자 지정 기간 설정
- ✅ **상태 필터**: 전체/진행중/완료/취소 상태별 필터링
- ✅ **실시간 데이터**: 송파지점, VIP 고객 등 실제 운영 데이터 표시

### **3. UI/UX**
- ✅ **반응형 디자인**: 모바일 퍼스트, 데스크톱 호환
- ✅ **로딩 상태**: API 호출 중 로딩 표시
- ✅ **에러 핸들링**: 사용자 친화적 오류 메시지
- ✅ **D'LIVE 브랜딩**: 기업 아이덴티티 반영

## 🔗 API 연동 현황

### **연동 완료된 API (3개)**
| API | 엔드포인트 | 상태 | 설명 |
|-----|-----------|------|------|
| 로그인 | `POST /api/login` | ✅ 완료 | 실제 딜라이브 계정 인증 |
| 작업조회 | `POST /api/work/directions` | ✅ 완료 | 실제 DB에서 작업지시 목록 조회 |
| 작업취소 | `POST /api/work/cancel` | ✅ 완료 | 실제 작업취소 처리 |

### **API 서버 정보**
- **서버**: `http://58.143.140.222:8080`
- **인증**: 세션 쿠키 기반 (`JSESSIONID`)
- **데이터**: 실제 운영 DB 연동 (더미 데이터 아님)

## 🚀 기술 스택

- **Frontend**: React 18 + TypeScript + Vite
- **스타일링**: Tailwind CSS
- **API 통신**: Fetch API + CORS
- **상태관리**: React Hooks (useState, useEffect)
- **빌드**: Vite (개발/배포)

- 🔄 **상담이력**: 고객 상담 내용 등록

## 🏗️ 아키텍처

### **시스템 구조**
```
Mobile CONA Frontend (React)
    ↓ JSON API 호출
딜라이브 JSON API 서버 (Spring)
    ↓ 서비스/DAO 호출
딜라이브 운영 DB (실제 데이터)
```

### **기존 시스템과의 관계**
- **기존**: MiPlatform Client ↔ MiPlatform Server (*.req)
- **신규**: React Frontend ↔ JSON API Server (/api/*)
- **공존**: 두 시스템이 동일한 서비스/DAO/DB를 공유하여 병행 운영
